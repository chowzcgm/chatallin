# Contributing to chatallin

Thanks for your interest in contributing to chatallin.

## Ground Rules

- Be respectful and constructive
- Keep pull requests focused and small
- Document behavior changes
- Avoid breaking existing interfaces without discussion

## Development Workflow

1. Fork the repository
2. Create a branch from `main`
3. Make your changes
4. Run local checks/tests (when available)
5. Open a pull request with clear context

## Commit Convention

Recommended commit prefixes:

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `refactor:` code refactor
- `test:` tests
- `chore:` maintenance

## Pull Request Checklist

- Describe what changed and why
- Link related issue (if any)
- Mention any breaking change explicitly
- Include test notes or reproduction steps

## Reporting Issues

Use the issue templates and include:

- Environment information
- Steps to reproduce
- Expected behavior
- Actual behavior

## Security

Please do not open public issues for sensitive vulnerabilities.

For now, contact the maintainer directly through GitHub profile.
